# Othello ローカル対戦テスト仕様書

## 1. 文書情報

| 項目 | 内容 |
|---|---|
| 文書目的 | 旧版AIと新版AIをローカルで対戦させるテスト環境の仕様を定義する |
| 対象BAT | `test/run_othello_matches.bat` |
| 対象レフェリー | `test/src/othello_match_runner.py` |
| 対象OS | Windows |
| 対応AI | CodinGame Othello形式で標準入出力を行うC++またはPythonプログラム |
| 画面表示 | なし。結果はコンソールへテキスト出力する |

## 2. 目的

本テスト環境は、AI修正前の旧版と修正後の新版を同じ条件で対戦させ、次の情報を確認するために使用します。

- 各局の勝敗
- 各AIが担当した色
- 終局時の石数
- 指定した対局数における勝数
- 指定した対局数における総石数
- 旧版と新版の総合判定

盤面を描画するGUIや人間が着手する操作機能は対象外です。

## 3. ファイル構成

```text
test/
├─ run_othello_matches.bat
├─ build/
│  ├─ old_bot.exe
│  └─ new_bot.exe
└─ src/
   └─ othello_match_runner.py
```

| ファイル・ディレクトリ | 役割 |
|---|---|
| `run_othello_matches.bat` | パス入力、環境確認、コンパイル、レフェリー起動を行う |
| `othello_match_runner.py` | 盤面管理、AIプロセス管理、時間監視、勝敗判定、集計を行う |
| `test/build/` | C++版AIの一時実行ファイルを配置する |
| `old_bot.exe` | 旧版C++ AIのコンパイル結果 |
| `new_bot.exe` | 新版C++ AIのコンパイル結果 |

## 4. 起動仕様

リポジトリルートで、引数を付けずにBATを実行します。

```powershell
.\test\run_othello_matches.bat
```

BATは次の順番で相対パス、新版の変更内容、対局数の入力を要求します。

```text
old_cpp_file: src\cpp\Othello_world_cup_ver_1.cpp
new_cpp_file: src\cpp\Othello_world_cup_ver_2.cpp
Change_Details: Remove last move position score from evaluation
Games: 100
```

| 入力 | 内容 |
|---|---|
| `old_cpp_file` | 旧版AIファイルのリポジトリルートからの相対パス |
| `new_cpp_file` | 新版AIファイルのリポジトリルートからの相対パス |
| `Change_Details` | 新版AIだけに加えた変更内容 |
| `Games` | 実行する対局数。1以上の整数 |

ファイルパスを囲むダブルクォートは除去されます。いずれかが空入力、または入力したファイルが存在しない場合、エラーを表示して終了します。

## 5. 対応ファイル

### 5.1 C++ファイル

拡張子が`.cpp`の場合、C++17、最適化レベル`O2`でコンパイルします。

```text
g++ -std=c++17 -O2 入力ファイル -o 出力ファイル
```

コンパイラは次の優先順位で使用します。

1. `C:\msys64\ucrt64\bin\g++.exe`が存在する場合は、そのコンパイラを使用する。
2. 存在しない場合は、環境変数`PATH`から`g++`を検索する。

MSYS2版GCCを使用するときは、CondaなどとのDLL競合を避けるため、`C:\msys64\ucrt64\bin`をBATプロセス内の`PATH`先頭へ設定します。

日本語を含む絶対出力パスによるMSYS2リンカーの失敗を避けるため、コンパイル時の出力先にはリポジトリルート基準の相対パスを使用します。

### 5.2 Pythonファイル

拡張子が`.py`の場合はコンパイルせず、レフェリーと同じPython実行環境で起動します。

### 5.3 非対応ファイル

`.cpp`と`.py`以外の拡張子はエラーとして扱います。

## 6. 対戦数と色

- 1回のBAT実行で指定した対局数を行います。
- 奇数局では旧版が黒、新版が白です。
- 偶数局では新版が黒、旧版が白です。
- 対局数が偶数の場合、旧版と新版は黒と白をそれぞれ同数担当します。
- 対局数が奇数の場合、旧版が黒を1局多く担当します。
- 各局ごとに旧版AIと新版AIのプロセスを新しく起動します。
- 各局は標準初期盤面から開始します。

| 局番号 | 黒 | 白 |
|---:|---|---|
| 奇数局 | OLD | NEW |
| 偶数局 | NEW | OLD |

## 7. ゲームルール

ローカルレフェリーは、CodinGame Othelloの共通ルールに従って次を処理します。

1. 8×8盤面を使用する。
2. 黒をプレイヤー`0`、白をプレイヤー`1`とする。
3. 黒から着手を開始する。
4. 縦、横、斜めの8方向で挟んだ相手石を反転する。
5. 1枚以上の相手石を反転できる空きマスだけを合法手とする。
6. 合法手がないプレイヤーは自動的にパスし、AIプロセスへ入力を送らない。
7. 両者に合法手がない場合は終局とする。
8. 終局時に石数が多いAIを勝者とする。
9. 石数が同じ場合は引き分けとする。

詳細なゲームルールは[CodinGame Othello 公式共通ルール・入出力仕様書](CodinGame_Othello_Official_Common_Rules_Specification.md)を正とします。

## 8. AI入出力仕様

### 8.1 初期入力

AIプロセス起動後、次を標準入力へ送ります。

```text
playerId
8
```

### 8.2 ターン入力

AIに合法手があるターンだけ、次を送ります。

```text
盤面8行
合法手数
合法手
...
```

盤面文字は空きマス`.`、黒`0`、白`1`です。合法手は`a1`から`h8`までの座標で、1行に1手を送ります。

### 8.3 EXPERTモード

AIが`EXPERT 座標`を出力した場合、そのAIでは次回以降の入力で盤面8行と合法手数の間に相手履歴を追加します。相手の連続着手と自動パスは、CodinGame仕様と同じくセミコロン区切りで送ります。

### 8.4 AI出力

次の出力を受け付けます。

```text
d3
d3 MSG message
EXPERT d3
EXPERT d3 MSG message
```

座標は小文字へ正規化し、当該ターンに送信した合法手一覧に含まれることを確認します。

## 9. 時間制限

| 対象 | 制限時間 |
|---|---:|
| 各AIの最初の行動 | 2.0秒 |
| 各AIの2回目以降の行動 | 0.15秒 |

計測範囲は、レフェリーがターン入力を書き始めてから、AIの標準出力を1行受信するまでです。制限時間を超えたAIは、その局を時間切れ負けとします。

## 10. 異常時の敗北条件

次の場合、該当AIをその局の敗者とします。

| 理由 | 内容 |
|---|---|
| `TIMEOUT` | 制限時間内に出力しなかった |
| `PROCESS_ERROR` | AIの標準入力へ書き込めなかった |
| `PROCESS_EXITED` | 着手出力前にAIプロセスが終了した |
| `EMPTY_OUTPUT` | 空行を出力した |
| `INVALID_OUTPUT` | 解釈できない形式を出力した |
| `PASS_OUTPUT` | 合法手があるターンに`PASS`を出力した |
| `ILLEGAL_MOVE` | 送信した合法手一覧にない座標を出力した |

異常終了した局の石数には、異常が発生した時点の盤面上の石数を表示します。勝敗は石数にかかわらず、異常を発生させたAIの敗北です。

## 11. 局ごとの出力

対局開始前に、新版の変更内容と対戦数を表示します。

```text
Match Information
NEW Change: Remove last move position score from evaluation
Games: 100
```

通常の勝敗は次の形式で表示します。

```text
Game 1
Winner: NEW
OLD: Color=Black, Stones=28
NEW: Color=White, Stones=36
```

異常による敗北では、追加で理由を表示します。

```text
Reason: TIMEOUT
```

引き分けの場合は次の形式です。

```text
Game 1
Result: Draw
OLD: Color=Black, Stones=32
NEW: Color=White, Stones=32
```

各局の結果の後には空行を1行出力します。

## 12. 最終集計と総合判定

指定した対局数の終了後、各AIの勝数、総石数、引き分け数を表示します。

```text
Final Summary
OLD: Wins=3, Total Stones=274
NEW: Wins=7, Total Stones=360
Draws: 0
```

総合判定の優先順位は次のとおりです。

1. 勝数が多いAIを優れていると判定する。
2. 勝数が同じ場合は、指定した全対局の総石数が多いAIを優れていると判定する。
3. 勝数と総石数が同じ場合は`TIE`とする。

## 13. 制約事項

- 対戦数は1以上の整数を指定できます。
- 初期局面のランダム化は行いません。
- GUIによる盤面表示は行いません。
- AIの標準エラーは読み捨て、通常の結果画面には表示しません。
- OSの負荷やプロセススケジューリングにより、同じAI同士でも探索到達深度や石数が変わる場合があります。
- 本環境の時間計測はローカルPC上の実時間であり、CodinGame実行環境と完全に同じ性能を保証するものではありません。

## 14. 受入条件

1. BATが引数なしで起動できること。
2. 旧版と新版の相対パスを対話入力できること。
3. C++ファイルをC++17、`O2`でコンパイルできること。
4. 奇数局と偶数局で黒白が交代すること。
5. 指定した対局数すべての結果が表示されること。
6. 各局に勝者、敗者、色、石数が表示されること。
7. 自動パスと終局を正しく処理すること。
8. 不正手、異常終了、時間切れを敗北として処理すること。
9. 最終集計と総合判定が表示されること。
